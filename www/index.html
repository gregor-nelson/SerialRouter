Production-Hardened Features

Built for unattended 24/7 operation

Automatic Recovery

Exponential backoff retry logic with watchdog thread monitoring. Automatically restarts failed connections.

Advanced Monitoring

Real-time throughput tracking, queue latency, MTBF calculations, uptime percentages, and connection health.

Thread-Safe Architecture

Centralized PortManager prevents conflicts. Three dedicated routing threads with exclusive port ownership.

Bidirectional Routing

Full duplex communication with return paths. Applications can read from and write back to the source.

GUI & Headless

PyQt6 interface with ribbon toolbar and real-time diagram, or run headless as a background service.

Intelligent Port Detection

Auto-detects and classifies ports (Physical, Moxa, com0com). Prevents configuration errors.

How It Works

1→2 routing architecture for offshore reliability

Core Components

  • • PortManager: Centralized port access with thread-safe operations
  • • Routing Threads: Dedicated threads with exclusive port ownership
  • • Watchdog Monitor: Automatic thread health checks and restarts
  • • Queue System: Buffered data distribution with latency tracking

Reliability Features

  • • Auto-Recovery: Exponential backoff handles connection failures
  • • Error Isolation: Port failures don't affect system operation
  • • Memory Safe: Counter resets prevent long-term memory growth
  • • Rotating Logs: 10MB log rotation for system health tracking

Monitoring Capabilities

  • • Throughput: Real-time bytes/sec with 60-second rolling window
  • • MTBF: Mean Time Between Failures calculations
  • • Uptime: Connection availability percentage tracking
  • • Queue Latency: Data flow timing and bottleneck detection

Primary Use Case

  • • Offshore Equipment: Single serial output feeding multiple systems
  • • Marine Installations: Reliable routing in remote environments
  • • Dual Applications: Enable two apps to read same data source
  • • 24/7 Operation: Memory leak prevention and automatic recovery

Requirements & Setup

Open source software with minimal dependencies

System Requirements

  • OS: Windows 10 or 11
  • Python: 3.7 or higher
  • Dependencies: PyQt6, pyserial
  • Resources: Minimal CPU and memory

Required: com0com

SerialRouter requires com0com virtual serial port driver to create paired ports.

  • • Free, open-source Windows driver
  • • Creates virtual COM port pairs
  • • Required before running SerialRouter
Download com0com

Quick Setup

1

Install com0com and create virtual port pairs (COM131/COM132, COM141/COM142)

2

Clone repository: git clone https://github.com/gregor-nelson/SerialRouter.git

3

Install dependencies: pip install -r requirements.txt

4

Run GUI: python main.py or Headless: python src/core/router_engine.py
